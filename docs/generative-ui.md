# Generative UI (soli-prof)

La **registry** (`lib/generative-ui/registry.ts`) è l’unica fonte di verità per i *render tool* esposti a Claude: nome in `snake_case` con prefisso `show_`, descrizione per il modello, `input_schema` JSON Schema compatibile con l’API Anthropic e funzioni di parse lato client. `getAnthropicTools()` produce l’array `tools` passato a `messages.create`; `isRenderTool` / `parseRenderToolInput` collegano il nome del tool ai props del componente React senza usare `any` nei punti pubblici.

Per **aggiungere un nuovo render tool** in tre passi: (1) crea un componente presentazionale sotto `components/generative-ui/` usando i token Tailwind del preset `@soli92/solids` (`bg-card`, `border-border`, ecc.); (2) estendi `lib/generative-ui/registry.ts` con un nuovo elemento in `RENDER_TOOLS_LIST`, lo schema e un parser dedicato, poi collegalo in `components/generative-ui/GenerativeToolMount.tsx`; (3) aggiorna il system prompt in `lib/prompts.ts` con quando usare il tool. Non definire tool inline nella route: la route importa solo `getAnthropicTools()`.

Il **parser dello stream** dopo il blocco `__SOURCES__` usa righe **NDJSON** (`lib/generative-ui/stream-protocol.ts`, campo `v: 1`): `text` con indice blocco e delta, `tbeg` / `tjson` / `tend` per i `tool_use`. Il client (`components/chat-view.tsx`) mantiene una `Map` per indice (`lib/generative-ui/merge-stream-slots.ts`), aggiorna l’UI a ogni riga completa e mostra uno skeleton finché non arriva `tend` per quel tool. Alla fine (`[DONE]`) chiama `finalizeStreamingSlots` per chiudere eventuali JSON incompleti. La cronologia inviata al server usa `buildAnthropicMessages` (`lib/generative-ui/to-anthropic-messages.ts`): dopo un assistente con `tool_use` di tipo render, viene inserito automaticamente un messaggio `user` con `tool_result` `"rendered"`, senza secondo turno modello nella stessa richiesta HTTP.
